## 核心策略：RAG 驱动 + 逐概念充实

**不是逐篇访谈处理，而是逐个概念处理**。对每个概念：

1. **RAG search** — 查询该概念在所有访谈中的段落
2. **RAG timeline** — 查看概念的时间线，找出访谈来源的条目
3. **筛选有价值的访谈内容** — 提取独特观点、原话、案例
4. **更新概念页** — 添加访谈来源的引用、原话、新视角
5. **建立反向链接** — 确保访谈页也链接到概念页

## 执行流程（每个概念）

```
对于每个概念 X:
1. uv run python -m rag search "X" → 获取所有访谈中的段落
2. uv run python -m rag timeline "X" → 查看时间线，筛选访谈来源
3. 对比现有概念页内容 → 识别增量信息
4. 如果有新内容:
   a. 在概念页添加「访谈中的观点」章节或补充到现有章节
   b. 添加访谈原话引用（格式：> "原话" — [访谈标题](链接)）
   c. 确保反向链接存在
5. 更新 index.md（如有新页面）
```
