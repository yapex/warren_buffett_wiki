## 三、每篇改造的工作清单

每篇案例改造需完成以下步骤：

### Step 1：资料收集（RAG 查询）
- [ ] 用 RAG 查询投资当年及后续年份信件中关于该公司的所有提及
- [ ] 提取巴菲特原话（当年解释 + 后续回顾 + 长期回顾）
- [ ] 收集关键数据（金额、比例、时间线）

### Step 2：Frontmatter 改造
- [ ] `source` 改为 `巴菲特亲笔信 + 原书籍名`
- [ ] 删除 `case_number` 字段
- [ ] 确认 `theme` 描述准确（2-6 字）

### Step 3：结构改造（六步法）

| 新章节 | 对应旧内容 | 改造动作 |
|--------|----------|---------|
| **一、决策场景还原** | ← `案例概述` + `投资背景` | 合并，增加"巴菲特的位置"子章节 |
| **二、关键数据** | ← 从各处提取 | 新建表格，投资决策 + 投资结果 |
| **三、巴菲特的决策过程** | ← `投资逻辑分析`（删除二手解读） | 替换为信件原话引用，按年份排列 |
| **四、决策对比点** | ← `学习要点`（删除预设结论） | 改为 4-6 个开放式问题 |
| **五、后续发展** | ← 散布在各处 | 新建时间线 + 巴菲特反思 |
| **六、延伸阅读** | ← `相关链接` | 规范化，增加相关信件列表 |

### Step 4：内容改造
- [ ] 删除 `投资哲学意义` 章节（预设结论）
- [ ] 删除 `与相关案例的对比` 章节（合并到延伸阅读）
- [ ] 删除 `原文摘录` 章节（融入"巴菲特的决策过程"）
- [ ] 删除 `附录` 章节（有价值的内容融入正文）
- [ ] 用自己的话重新表述（不直接复制书籍原文）
- [ ] 目标大小：10-20 KB

### Step 5：链接规范化
- [ ] 所有链接改为相对路径
- [ ] 文件名无空格（修复如 `盖可保险 -1951` → `盖可保险-1951`）
- [ ] 确认链接目标存在
- [ ] 增加相关信件链接列表

### Step 6：质量检查
- [ ] 对照 [质量标准检查清单](../../wiki/research/cases/README-案例研究规范.md) 逐项检查

| 1986-费希海默兄弟 | 内容严重不足（4 KB） | 需大量补充信件资料，如资料不足可考虑合并 |
| 1989-波仙珠宝 | 内容严重不足（4 KB） | 需大量补充信件资料 |

## 五、执行节奏建议

### 每篇预估工时：20-40 分钟

| 批次 | 案例数 | 预估总工时 | 建议 |
|------|--------|----------|------|
| 🔴 第一批 | 8 | 4-5 小时 | 核心案例，仔细打磨 |
| 🟡 第二批 | 11 | 5-6 小时 | 逐篇改造，保证质量 |
| 🟢 第三批 | 11 | 4-5 小时 | 可简化处理，重点在结构 |

### 建议执行方式

1. **每次改造 1-2 篇**，保证质量
2. **优先做第一批**，这些案例读者最多
3. **改造完成后更新此计划**，标记完成状态
4. **全部完成后**，更新 README-案例研究规范 增加更多参考范例
