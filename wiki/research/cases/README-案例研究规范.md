# 案例研究（Case Study）编写规范

> **文档目的**：记录 `wiki/research/cases/` 目录下投资案例研究的结构规范，方便后续生成新案例时参考。
> 
> **最后更新**：2026-04-09
> **适用范围**：基于《巴菲特的第一桶金》《巴菲特的伯克希尔崛起》等书籍的投资案例研究

---

## 一、文件命名规范

### 格式

```
公司名 - 年份 - 主题.md
```

### 示例

```
✅ 盖可保险 -1976-危机与复苏.md
✅ 美国运通 -1964-色拉油危机.md
✅ 可口可乐 -1988-消费巨头.md
✅ 内布拉斯加家具城 -1983-家族企业.md
✅ 所罗门兄弟 -1987-投行危机.md
```

### 规则

| 规则 | 说明 |
|------|------|
| **无空格** | 使用连字符 `-` 分隔，不能用空格 |
| **年份格式** | 4 位数字，如 `1976`；跨年份用 `1962-1965` |
| **主题简洁** | 2-6 个字概括案例类型，如 `危机与复苏`、`特许经营权` |
| **公司名简化** | 去掉冗余词，如 `公司`、`集团`（除非必要） |

## 二、Frontmatter 结构

```yaml
---
type: case_study
company: 公司名（可含英文）
year: 年份
source: 来源书籍
case_number: 案例编号
theme: 投资主题
---
```

### 字段说明

| 字段 | 必填 | 示例 | 说明 |
|------|------|------|------|
| `type` | ✅ | `case_study` | 固定为 `case_study` |
| `company` | ✅ | `盖可保险 (GEICO)` | 公司名，可含英文缩写 |
| `year` | ✅ | `1976` | 投资年份 |
| `source` | ✅ | `巴菲特的第一桶金` 或 `巴菲特的第一桶金 + 巴菲特的伯克希尔崛起` | 来源书籍 |
| `case_number` | ✅ | `2` | 原书中的案例编号 |
| `theme` | ✅ | `危机中的品质投资` | 投资主题/类型 |

### 完整示例

```yaml
---
type: case_study
company: 盖可保险 (GEICO)
year: 1976
source: 巴菲特的第一桶金 + 巴菲特的伯克希尔崛起
case_number: 2
theme: 危机中的品质投资
---
```

## 四、各章节详细说明

### 1. 标题与参考来源

```markdown

> **参考来源**：《书名》案例 X（作者著）
```

**示例**：
```markdown

```

### 2. 案例概述

**目的**：1-2 段话概括整个案例，让读者快速了解核心。

**内容**：
- 投资背景（危机/机会）
- 巴菲特的行动
- 最终结果/回报

**示例**：
```markdown
## 案例概述

1976 年，盖可保险因严重低估准备金和激进扩张濒临破产，股价从 61 美元暴跌至 2 美元（-97%）。巴菲特透过危机看到公司低成本直销模式的根本优势依然完好，在杰克·拜恩出任 CEO 后大举投资 4700 万美元。这笔投资最终增值超过 100 倍，被巴菲特称为"或许是最佳的单笔投资"。
```

### 4. 投资背景

**目的**：说明投资发生的市场和公司环境。

**子章节**：
- **市场环境**：行业状况、宏观经济
- **公司状况**：公司面临的问题/机会

**示例**：
```markdown
## 投资背景

### 市场环境

1970 年代中期美国保险行业遭遇多重打击：
- 行业综合成本率恶化（1975 年达 108.3）
- 26 个州通过《无过错保险法》
- 医疗和汽车维修成本快速上升

### 公司危机根源

1. **战略迷失**
   - 新管理层追求规模扩张
   - 取消职业限制，向所有人开放投保
   
2. **准备金严重低估**
   - 1975 年发现低估了 1 亿美元理赔准备金
```

---

### 5. 巴菲特的调研/分析

**目的**：展示巴菲特如何分析这个机会。

**子章节**：
- **调研方法**：如何获取信息（实地调研、历史认知、类比分析等）
- **关键发现**：分析得出的核心洞察
- **核心结论**：最终的投资判断

**示例**：
```markdown
## 巴菲特的调研与分析

### 调研方法

**历史认知积累**：
- 1951 年首次投资盖可（20 岁，65% 净资产）
- 25 年持续关注公司发展

**关键人物评估**：
- 通过凯瑟琳·格雷厄姆安排与杰克·拜恩会面
- 评估拜恩的能力、品格和拯救计划

### 关键发现

1. **危机性质判断**
   - 不是根本性经济优势的丧失
   - 是管理层短期失误导致的可修复问题

2. **管理层评估**
   - 杰克·拜恩具备拯救公司所需的所有素质
```

---

### 6. 投资决策与执行

**目的**：记录投资的具体执行过程。

**子章节**：
- **关键时点**：时间线表格
- **仓位管理**：投资金额、持股比例、投资结构

**示例**：
```markdown
## 投资决策与执行

### 关键时点

| 时间 | 事件 |
|------|------|
| 1976 年 5 月 | 杰克·拜恩出任 CEO |
| 1976 年 7 月 | 伯克希尔买入 129 万股普通股（411 万美元） |
| 1976 年 11 月 | 优先股发行完成 |
| 1980 年 | 持股比例达 33% |

### 投资结构

**普通股投资**：
- 1976 年 7 月：411.6 万美元，129 万股

**优先股投资**：
- 1976 年 11 月：1942 万美元，198 万股
```

### 7. 投资逻辑分析

**目的**：深度分析投资背后的逻辑。

**子章节**（根据案例选择）：
- **格雷厄姆 vs 费雪/芒格标准**：对比分析
- **关键洞察**：核心投资逻辑
- **竞争优势分析**：护城河分析

**示例**：
```markdown
## 投资逻辑分析

### 格雷厄姆 vs 费雪/芒格标准

| 标准 | 类型 | 符合度 |
|------|------|--------|
| 低价买入 | 格雷厄姆 | ✅ 危机导致低估 |
| 安全边际 | 格雷厄姆 | ✅ 远低于内在价值 |
| 商业优势 | 费雪 | ✅ 低成本模式完好 |
| 管理层 | 费雪/芒格 | ✅ 杰克·拜恩卓越 |

### 关键洞察

1. **区分"暂时危机"与"永久损害"**
   - 危机中要问：根本优势是否还在？
   
2. **管理层是 turnaround 的关键**
   - 找到"技艺高超的外科医生"
```

### 8. 投资哲学意义

**目的**：将案例置于巴菲特投资哲学演变的大背景下。

**子章节**：
- **在投资哲学演变中的位置**：表格展示
- **与前后案例的关联**：相关链接
- **影响因素**：费雪、芒格等影响

**示例**：
```markdown
## 投资哲学意义

### 在投资哲学演变中的位置

| 时期 | 特点 | 本案例定位 |
|------|------|-----------|
| 1951-1960s | 格雷厄姆式低价股 | 首次投资盖可（1951） |
| 1976 | 危机中的品质投资 | 盖可 1976 年投资 |
| 1980s | 经济特许权 | 盖可作为永久性持股 |
```

## 五、版权注意事项 ⚠️

### 核心原则

**❌ 禁止**：
- 直接复制书籍原文超过 1-2 段
- 大段引用未加引号
- 原样照搬章节结构

**✅ 允许**：
- 用自己的话重新表述核心观点
- 引用关键数据（事实不受版权保护）
- 简短引用（1-2 句）加引号并注明

### 正确做法

1. **阅读原书章节** → 理解核心逻辑
2. **合上书本** → 用自己的话总结
3. **核对关键数据** → 确保准确性
4. **添加简短引用** → 巴菲特的原话可引用

### 文件大小参考

| 类型 | 大小范围 | 说明 |
|------|---------|------|
| ❌ 直接复制 | 40-70 KB | 应避免 |
| ✅ 重新表述 | 8-15 KB | 目标范围 |

## 八、生成流程

### 从书籍生成新案例的步骤

```
1. 确定案例
   └─> 从书籍目录找到目标案例
   
2. 阅读原章节
   └─> 理解核心投资逻辑
   
3. 提取关键数据
   └─> 时间、金额、持股比例、回报等
   
4. 编写 Frontmatter
   └─> 填写 6 个必填字段
   
5. 撰写案例概述
   └─> 1-2 段概括全文
   
6. 填充各章节
   └─> 按标准结构用自己的话表述
   
7. 添加相关链接
   └─> 链接到公司、信件、概念页面
   
8. 质量检查
   └─> 对照检查清单
   
9. 保存文件
   └─> 正确命名，无空格
```

## 附录：常见问题

### Q: 同一案例在两本书中都有，如何处理？

**A**：合并到一个文件中，在 Frontmatter 的 `source` 字段注明两本书：

```yaml
source: 巴菲特的第一桶金 + 巴菲特的伯克希尔崛起
```

在参考来源中说明：
```markdown
> **参考来源**：《巴菲特的第一桶金》案例 2 + 《巴菲特的伯克希尔崛起》案例 1
```

### Q: 如何判断是否需要新建案例页面？

**A**：以下情况应新建：
- 独立的投资事件
- 有明确的投资年份
- 有可量化的投资数据
- 有清晰的投资逻辑可总结

### Q: 文件大小应该控制在多少？

**A**：目标 8-15 KB。超过 20 KB 通常意味着复制了过多原文，需要精简。
