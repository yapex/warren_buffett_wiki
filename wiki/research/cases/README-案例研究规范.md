# 案例研究（Case Study）编写规范

> **文档目的**：记录 `wiki/research/cases/` 目录下投资案例研究的结构规范，方便后续生成新案例时参考。
> 
> **最后更新**：2026-04-09
> **核心理念**：案例不是用来"学习答案"的，而是用来"练习决策"的

---

## 一、文件命名规范

### 格式

```
年份-公司名-主题.md
```

### 示例

```
✅ 1976-盖可保险-危机与复苏.md
✅ 1964-美国运通-色拉油危机.md
✅ 1988-可口可乐-消费巨头.md
✅ 1983-内布拉斯加家具城-家族企业.md
✅ 1987-所罗门兄弟-投行危机.md
✅ 1962-1965-伯克希尔哈撒韦-从投资到控股.md
```

### 规则

| 规则 | 说明 |
|------|------|
| **年份在前** | 便于按时间排序，具体年份或年份范围（如 `1962-1965`） |
| **无空格** | 使用连字符 `-` 分隔，**不能用空格** |
| **主题简洁** | 2-6 个字概括案例类型，如 `危机与复苏`、`特许经营权` |
| **公司名简化** | 去掉冗余词，如 `公司`、`集团`（除非必要） |

## 二、Frontmatter 结构

```yaml
---
type: case_study
company: 公司名（可含英文）
year: 年份
source: 来源
theme: 投资主题
---
```

### 字段说明

| 字段 | 必填 | 示例 | 说明 |
|------|------|------|------|
| `type` | ✅ | `case_study` | 固定为 `case_study` |
| `company` | ✅ | `盖可保险 (GEICO)` | 公司名，可含英文缩写 |
| `year` | ✅ | `1976` | 投资年份 |
| `source` | ✅ | `巴菲特亲笔信 + 巴菲特的第一桶金` | **一手资料优先**，信件 + 书籍 |
| `theme` | ✅ | `危机中的品质投资` | 投资主题/类型 |

### 来源字段规范

**旧方式（不推荐）**：
```yaml
source: 巴菲特的第一桶金
case_number: 2
```

**新方式（推荐）**：
```yaml
source: 巴菲特亲笔信 + 巴菲特的第一桶金 + 巴菲特的伯克希尔崛起
```

**原则**：
- ✅ **巴菲特亲笔信优先**：这是第一手资料，最真实
- ✅ **多来源交叉验证**：信件 + 书籍 + 其他资料
- ❌ **不再使用 case_number**：这是书籍章节编号，不重要

## 三、内容结构

### 核心理念转变

| 维度 | 旧设计 | 新设计 |
|------|--------|--------|
| **目的** | 学习巴菲特的结论 | 练习自己的决策 |
| **来源** | 二手书籍解读 | 巴菲特亲笔信原话 |
| **结论** | 给答案（学习要点） | 提问题（决策对比点） |
| **框架** | 格雷厄姆 vs 费雪分类 | 还原决策场景 |

## 四、各章节详细说明

### 1. 决策场景还原

**目的**：让读者置身于巴菲特当时的决策环境。

**子章节**：
- **时间与市场环境**：行业状况、宏观经济、市场情绪
- **公司危机/机会**：公司面临的问题或机会，关键数据
- **巴菲特的位置**：他的年龄、经验、资金状况、当时持仓

**参考范例**：[1976-盖可保险-危机与复苏](1976-盖可保险-危机与复苏.md)

---

### 2. 关键数据

**目的**：量化展示投资决策和结果。

**子章节**：
- **投资决策**：时间、金额、持股比例、投资结构
- **投资结果**：市值变化、回报倍数、持有期

---

### 3. 巴菲特的决策过程

**目的**：展示巴菲特**自己的话**，而非作者的解读。

**关键原则**：
- ✅ **引用当年信件**：他向股东解释的投资逻辑
- ✅ **引用后续回顾**：他几年后的反思和总结
- ✅ **引用长期回顾**：20 年后的最终评价（如有）
- ❌ **不引用二手书籍**：那是作者的解读，不是巴菲特的原话

**子章节按时间排列**：
- **投资当年的解释**：YYYY-letter 中的原话
- **几年后的回顾**：YYYY+4-letter 或 YYYY+10-letter 中的反思
- **长期回顾**：YYYY+20-letter 或更晚的评价（如有）

---

### 4. 决策对比点

**目的**：不给预设结论，引导读者思考"**如果是我，会怎么做？**"

**六个维度**：
1. **信息层面**：哪些信息公开可得？哪些需要主动调研？
2. **时机层面**：你会等到什么时候买入？
3. **仓位层面**：你会下多大注？
4. **风险层面**：你担心什么风险？
5. **持有层面**：你会什么时候卖出？
6. **核心判断**：你如何区分"暂时危机"和"永久损害"？

每个维度包含：
- **实际发生**：巴菲特做了什么
- **问题**：开放式问题，引导读者独立思考

---

### 5. 后续发展

**目的**：记录投资后的实际结果和巴菲特的反思。

**子章节**：
- **时间线**：关键事件的时间序列
- **巴菲特的反思**：他对这笔投资的回顾（引用后续信件）

---

### 6. 延伸阅读

**目的**：建立 Wiki 内部链接网络，引导读者深入研究。

**子章节**：
- **相关信件**：按时间顺序列出所有相关信件
- **相关案例**：同一公司、同一类型、同一时期的其他案例
- **相关概念**：本案例涉及的投资概念
- **相关人物**：本案例涉及的关键人物

---

## 五、删除的章节

以下章节在新设计中**不再使用**：

| 旧章节 | 删除原因 |
|--------|---------|
| `## 案例概述` | 合并到"决策场景还原" |
| `## 投资背景` | 合并到"决策场景还原" |
| `## 巴菲特的调研/分析` | 改为"巴菲特的决策过程"，强调引用原话 |
| `## 投资决策与执行` | 合并到"关键数据" |
| `## 投资逻辑分析` | 删除预设框架（格雷厄姆 vs 费雪） |
| `## 投资哲学意义` | 删除预设结论 |
| `## 学习要点` | 改为"决策对比点"，不给答案 |

## 六、版权与引用规范

### 核心原则

**✅ 必须引用巴菲特原话**：
- 当年信件中的解释
- 后续信件中的回顾
- 简短引用加引号并注明来源信件

**✅ 用自己的话重新表述**：
- 阅读信件原文 → 理解核心逻辑
- 合上书本 → 用自己的话总结
- 添加"决策对比点"引导思考

**❌ 禁止**：
- 直接复制书籍原文超过 1-2 段
- 大段引用未加引号
- 原样照搬二手作者的解读
- 中英文混用（如"成本结构 intact"应改为"成本结构完好"）

### 文件大小参考

| 类型 | 大小范围 | 说明 |
|------|---------|------|
| ❌ 直接复制 | 40-70 KB | 应避免 |
| ❌ 二手解读 | 15-20 KB | 应避免 |
| ✅ 重新表述+原话引用 | 10-20 KB | 目标范围 |

## 八、质量标准

### 内容检查清单

- [ ] Frontmatter 完整（source 包含信件）

## 九、生成流程

### 从信件生成新案例的步骤

```
1. 确定案例
   └─> 从信件中找到投资事件（持仓披露、重大投资说明）
   
2. 用 RAG 收集相关信件
   └─> 投资当年信件（解释逻辑）
   └─> 后续年份信件（回顾反思）
   └─> 长期回顾信件（最终评价）
   
3. 提取关键数据
   └─> 时间、金额、持股比例、回报等
   
4. 编写 Frontmatter
   └─> source: 巴菲特亲笔信 + 其他资料
   
5. 撰写决策场景还原
   └─> 还原当时的市场、公司、巴菲特位置
   
6. 引用巴菲特原话
   └─> 按时间顺序排列（当年 → 后续 → 长期）
   
7. 设计决策对比点
   └─> 4-6 个开放式问题，引导读者思考
   
8. 添加延伸阅读
   └─> 链接到信件、案例、概念、人物
   
9. 质量检查
   └─> 对照检查清单
```
